# NetPractice

### Description
This project gives us a general understanding of computer networking, dealing with subjects such as IP-address configuration, subnet masking, router connections, and TCP and UDP protocols.

### Instructions
- First, download the file attached to the project’s page.
- Then, extract the files into any folder of your choice.
- In this folder, run the ```run.sh``` file. This shell script will launch a web server and
open your preferred web browser to the dedicated page.
- The NetPractice interface will open in your browser and you'll be given two options: Training and Evaluation modes.
- In order to dowload a configuration, click on the _Get my config_ button.

### Sumission details
The 10 successful configurations to the levels provided in the NetPractice interface must have been dowloaded and added to the root of this repository. Their names are ```level<level number>.json```.

### Resources
For this project, the subjects studied were:

- IP addressing
- Subnet masks
- CIDR notation
- Interfaces
- Gateways
- Switches
- Broadcasting
- Routers and routing tables
- TCP and UDP protocols

The sources used for this purpose were:

- Beej's Guide to Network Concepts, sections 6 and 7: https://beej.us/guide/bgnet0/html/split/index.html
- CCNA course by NetworkChuck: https://www.youtube.com/watch?v=S7MNX_UD7vY
- Video on subnet masking by PowerCert Animated Videos: https://www.youtube.com/watch?v=s_Ntt6eTn94
- Video on default gateways by PowerCert Animated Videos: https://www.youtube.com/watch?v=pCcJFdYNamc

AI was used to inquire about general networking doubts rather than to look into specifics of the project, e. g., to understand why a routing table is not necessary when needing to send packets to directly-connected networks.
